# Shadows

Shadows in design are used to create depth and hierarchy. They can be used to differentiate between different elements, and to highlight interactive elements.

<Canvas of={ShadowStories.DefaultStory} />

## Size

Shadows come in four different sizes: XS, SM, MD, and LG. The size of the shadow can be used to indicate the relative importance or prominence of an element.

For Tailwind web consumers, the default neutral shadows are available directly as utility classes:

- `shadow-xs`
- `shadow-sm`
- `shadow-md`
- `shadow-lg`

## Color

Shadows can have three different colors: default (neutral), primary, and error/danger. The color of the shadow can be used to indicate the state or functionality of an element.

Default neutral shadows are covered by the standard Tailwind shadow size utilities above. When you need a colored shadow on the web, compose the shadow size token with the shadow color token.

## Usage

Shadows can be applied to various components to create depth and hierarchy. Here are some examples of how shadows can be used in different components:

- Use `shadow-xs|sm|md|lg` for standard neutral shadows in Tailwind.
- Use CSS variables when you need an explicit colored shadow composition.

**NOTE: The CSS-in-JS `shadows.size` objects for React Native contain all the correct tokens for neutral shadows. For primary and error/danger shadows change the `shadowColor` key**

Example shape of the `xs` shadow size object from `shadows`

```
size: {
    xs: {
      shadowColor: colors.light.shadow.default,
      shadowOffset: {
        width: 0,
        height: 2,
      },
      shadowOpacity: 1,
      shadowRadius: 4,
    },
    ...
}

```
