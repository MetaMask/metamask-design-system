# Skeleton

`Skeleton` displays a pulsing placeholder while content is loading. It mirrors the React Native `Skeleton` API on `@metamask/design-system-react-native`, so consumers can use the same mental model across platforms.

```tsx
import { Skeleton } from '@metamask/design-system-react';

// Standalone placeholder (no children) — always renders the animated bar.
<Skeleton width={240} height={16} />;

// Wrapping content — renders the children directly by default; pass
// `hideChildren` to overlay the skeleton while children dictate layout.
<Skeleton hideChildren>
  <Text variant={TextVariant.HeadingMd}>This text dictates the skeleton's size</Text>
</Skeleton>;
```

## Props

### `width` / `height`

Optional CSS dimensions for the skeleton container. Numbers are interpreted as px, strings can be any CSS length (`'2rem'`, `'100%'`, etc.). When `children` is provided with `hideChildren`, the children's natural layout takes priority — `width` / `height` are useful for standalone placeholders.

### `hideChildren`

When `true`, children render invisibly while the skeleton overlay plays — children preserve layout dimensions but assistive tech and visible UI see only the skeleton. When `false` (default), children render directly with no skeleton overlay.

This is the **inverse** of the legacy extension `Skeleton`'s `isLoading` prop: `isLoading={true}` ≡ `hideChildren={true}`.

### `autoPlay`

When `true`, the skeleton plays the pulse animation. Set to `false` to render a static placeholder (useful in screenshots, golden-path tests, or when you want CSS-only fallbacks).

## Accessibility

The skeleton container and its animated overlay are both marked `aria-hidden="true"` and `pointer-events-none`. They take no part in the accessibility tree — assistive tech sees through to whatever real content is shown when loading completes.

When using `hideChildren`, the children are rendered inside an `aria-hidden` wrapper too, so they don't double-announce while the skeleton is showing.
