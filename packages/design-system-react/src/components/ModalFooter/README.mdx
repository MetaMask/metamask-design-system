# ModalFooter

`ModalFooter` is the bottom section of a modal. It hosts up to two action buttons — primary and secondary — and an optional slot above them for inline form controls (a "do not show again" checkbox, a disclaimer, etc.).

The component owns the button semantics: it renders the primary button with `ButtonVariant.Primary` and the secondary button with `ButtonVariant.Secondary` automatically. Consumers configure each via `primaryButtonProps` / `secondaryButtonProps`, but cannot override the variant. This mirrors the React Native `BottomSheetFooter` so the modal/sheet families share semantics across platforms.

```tsx
import { ModalFooter } from '@metamask/design-system-react';

<ModalFooter
  primaryButtonProps={{ children: t('confirm'), onClick: handleSubmit }}
  secondaryButtonProps={{ children: t('cancel'), onClick: handleClose }}
/>;
```

## Props

### `primaryButtonProps`

Props for the primary action button. Renders with `ButtonVariant.Primary` automatically. Appears second (right in horizontal layout, bottom in vertical layout).

Pass the localized label as `children` and wire the click handler via `onClick`. `ModalFooter` is i18n-agnostic — pass localized strings from your application's i18n layer (e.g. `t('confirm')`).

### `buttonsAlignment`

Layout direction for the action button row.

- `ButtonsAlignment.Horizontal` (default) — buttons share the row, each takes `flex-1`. Order: secondary, primary.
- `ButtonsAlignment.Vertical` — buttons stack with full width. Order: secondary on top, primary on bottom.

### `className`

Use the `className` prop to add custom CSS classes to the outer `<footer>` element. These classes are merged with the component's defaults via `twMerge`, so consumer utilities override the defaults (for example, `className="px-0"` wins over the internal `px-4`).
