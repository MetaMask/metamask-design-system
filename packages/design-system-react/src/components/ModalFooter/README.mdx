# ModalFooter

`ModalFooter` is the bottom section of a modal. It hosts the primary submit/cancel button row and provides a slot above it for inline form controls (e.g. a "do not show again" checkbox). Pass `onSubmit` / `onCancel` to render the built-in buttons; omit them and provide `children` if you need a fully custom footer.

```tsx
import { ModalFooter } from '@metamask/design-system-react';

<ModalFooter
  onSubmit={handleSubmit}
  submitButtonProps={{ children: t('confirm') }}
  onCancel={handleClose}
  cancelButtonProps={{ children: t('cancel') }}
/>;
```

### `submitButtonProps` / `cancelButtonProps`

Props forwarded to the underlying submit / cancel `Button`. Use `children` for the (localized) label and any other prop to customize variant, danger styling, icons, etc.

`ModalFooter` is i18n-agnostic — the design system intentionally does not pull strings from a translation context. Pass localized strings from your application's i18n layer:

```tsx
<ModalFooter
  onSubmit={handleSubmit}
  submitButtonProps={{ children: t('confirm') }}
  onCancel={handleClose}
  cancelButtonProps={{ children: t('cancel') }}
/>
```

When `children` is omitted the component falls back to the literal `'Confirm'` / `'Cancel'` so prototypes and Storybook render correctly without an i18n setup.

### `containerProps`

Props passed to the inner `Box` that wraps the submit/cancel buttons. Use this to override the default 360px max-width, change the background, or apply any other Box utility. The default `max-w-[360px]` is applied via className and is overridden when `containerProps.className` provides its own `max-w-*` utility (`twMerge` lets the consumer's class win).

### `className`

Use the `className` prop to add custom CSS classes to the outer `<footer>` element. These classes are merged with the component's defaults via `twMerge`, so consumer utilities override the defaults (for example, `className="px-0"` wins over the internal `px-4`).
