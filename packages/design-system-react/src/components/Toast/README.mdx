# Toast

Toast is a notification that slides up from the bottom of the screen. It is typically used to show post-confirmation information such as account switches, network changes, or transaction confirmations.

<Canvas of={ToastStories.Default} />

```tsx
import { Button, Toaster, toast } from '@metamask/design-system-react';

const Demo = () => {
  return (
    <>
      <Button
        onClick={() => {
          toast({
            title: 'Title is sentence case no period',
            description: "Description shouldn't repeat title. 1-3 lines.",
          });
        }}
      >
        Show Toast
      </Button>
      <Toaster />
    </>
  );
};
```

## Usage

Render the `Toaster` component once in your app, then create a toast by calling `toast(...)`.

The live examples below use direct `<Toast />` rendering where that makes the component API easier to scan in Storybook. Use the code snippets for the imperative `toast(...)` flow.

```tsx
import { Button, Toaster, toast } from '@metamask/design-system-react';

const Demo = () => {
  return (
    <>
      <Button
        onClick={() => {
          toast({
            title: 'Title is sentence case no period',
            description: "Description shouldn't repeat title. 1-3 lines.",
          });
        }}
      >
        Show Toast
      </Button>
      <Toaster />
    </>
  );
};
```

`<Toaster />` must be rendered exactly once. It subscribes to the `toast(...)` / `toast.dismiss()` API so it can be called from anywhere in your app, including before `<Toaster />` mounts.

Call `toast.dismiss()` to dismiss the currently visible toast.

Use `closeButtonProps` to access the close button element when you need to set a `data-testid` or override its accessibility label. With the `toast(...)` / `<Toaster />` flow the close button is shown because `<Toaster />` wires `onClose` internally; for direct `<Toast />` rendering, provide `onClose` to render it.

If `toast(...)` is called before `<Toaster />` mounts, the latest toast renders after mount. If `toast.dismiss()` is called before mount, it clears the pending toast.

### `title`

Use `title` for the primary message. Use `titleProps` to access the title `Text` element when you need to set a `data-testid`, accessibility props, or other `Text` overrides.

<Canvas of={ToastStories.Title} sourceState="none" />

```tsx
import { toast } from '@metamask/design-system-react';

toast({
  title: 'We will notify you.',
});
```

### `description`

Use `description` for supporting content below the title. Use `descriptionProps` to access the description `Text` element when you need to set a `data-testid`, accessibility props, or other `Text` overrides.

<Canvas of={ToastStories.Description} sourceState="none" />

```tsx
import { toast } from '@metamask/design-system-react';

toast({
  title: "Don't miss out",
  description: 'Enable notifications to stay informed on campaigns.',
});
```

### `severity`

Use `severity` to show a semantic default start icon. Use `iconProps` to access the default `Icon` element when you need to set a `data-testid`, accessibility props, or other icon overrides for a non-default severity.

Available severities:

- `ToastSeverity.Default` - No default leading icon
- `ToastSeverity.Success`
- `ToastSeverity.Warning`
- `ToastSeverity.Danger`

<Canvas of={ToastStories.Severity} sourceState="none" />

```tsx
import { toast, ToastSeverity } from '@metamask/design-system-react';

toast({
  title: 'Network changed',
  description: 'You are now connected to Linea.',
  severity: ToastSeverity.Success,
});
```

### `startAccessory`

Use `startAccessory` to replace the default severity icon with custom content such as an icon, avatar, or spinner. Use `children` and `childrenWrapperProps` when you need to render additional content below the description, or to access that content wrapper for a `data-testid` and other element-level overrides.

<Canvas of={ToastStories.StartAccessory} sourceState="none" />

```tsx
import { Icon, IconName, IconSize, toast } from '@metamask/design-system-react';

toast({
  title: 'Withdrawal pending',
  description: 'Your withdrawal is processing.',
  startAccessory: <Icon name={IconName.Clock} size={IconSize.Lg} />,
});
```

### `actionButtonOnClick`

Use `actionButtonOnClick` together with `actionButtonLabel` when the toast needs a secondary action. Use `actionButtonProps` to access the rendered button element when you need to set a `data-testid`, accessibility props, or other button overrides.

<Canvas of={ToastStories.ActionButton} sourceState="none" />

```tsx
import { toast } from '@metamask/design-system-react';

toast({
  title: 'Privacy policy update',
  description: 'Review how Consensys handles your data.',
  actionButtonLabel: 'Read more',
  actionButtonOnClick: () => {
    console.log('Action clicked');
  },
});
```

### `onClose`

Use `onClose` for a side-effect callback when the close button is clicked. With the `toast(...)` / `<Toaster />` flow the close button is shown because `<Toaster />` passes `onClose` internally; provide `onClose` when you need to run code on dismiss. For direct `<Toast />` rendering, `onClose` must be provided to render the close button.

<Canvas of={ToastStories.OnClose} sourceState="none" />

```tsx
import { toast, ToastSeverity } from '@metamask/design-system-react';

toast({
  title: 'Account switched',
  description: 'You are now using Account 2.',
  severity: ToastSeverity.Success,
  onClose: () => {
    console.log('Toast dismissed');
  },
});
```

## Toast Options

- `title`, `titleProps` - Main toast content and optional text props.
- `description`, `descriptionProps` - Optional secondary content and text props.
- `children`, `childrenWrapperProps` - Optional extra content rendered below the description.
- `actionButtonLabel`, `actionButtonOnClick`, `actionButtonProps` - Optional action button content and handler.
- `onClose` - Optional callback invoked when the close button is clicked. Also renders the close button on direct `<Toast />`.
- `closeButtonProps` - Optional non-behavioural props merged onto the close `ButtonIcon` when it is rendered.
- `startAccessory` - Optional leading accessory that overrides the severity icon.
- `severity` - Optional semantic state used to choose the default icon. Defaults to `ToastSeverity.Default`, which shows no icon.
- `iconProps` - Optional props merged onto the default severity `Icon`.
- `hasNoTimeout` - When `true`, the toast stays visible until dismissed. Defaults to `false` (auto-dismisses after 2750 ms).
- `className` - Optional CSS classes merged onto the toast surface via `twMerge`.
