# ModalFocus

`ModalFocus` traps keyboard focus inside its subtree while it is mounted. Use it to wrap modal, dialog, or popover content so screen-reader and keyboard-only users cannot tab outside the open layer. It is a thin wrapper around [`react-focus-lock`](https://github.com/theKashey/react-focus-lock) with a small API for initial focus, final focus, and focus restoration.

```tsx
import { ModalFocus } from '@metamask/design-system-react';

<ModalFocus>{modalContent}</ModalFocus>;
```

## Props

### `children`

The subtree to lock focus inside of. While `ModalFocus` is mounted, Tab cycles only between focusable elements within `children`.

### `restoreFocus`

If `true`, focus is restored to the element that triggered `ModalFocus` once it unmounts. Ignored when `finalFocusRef` is provided.

### `autoFocus`

If `true`, the first focusable element in `children` is focused on mount. Set to `false` if you want the trap active without taking focus.
