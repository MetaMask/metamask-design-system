# ModalContent

`ModalContent` is the centered dialog container that lives inside a `Modal`. It owns the close-on-overlay-click and close-on-Escape handlers, hands focus management to `ModalFocus` (using the focus refs/flags from `useModalContext`), and applies the standard `slide-up` entrance animation.

`ModalContent` reads its behavior from `useModalContext`, so it must be rendered inside `<Modal>`.

```tsx
import { Modal, ModalContent } from '@metamask/design-system-react';

<Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
  <ModalContent>{/* ModalHeader / ModalBody / ModalFooter */}</ModalContent>
</Modal>;
```

## Behavior controlled via `Modal`

`ModalContent` reads the following flags from `useModalContext` and reacts to them:

| Flag                      | Effect                                                                                            |
| ------------------------- | ------------------------------------------------------------------------------------------------- |
| `isClosedOnEscapeKey`     | When `true`, pressing Escape calls `onClose`.                                                      |
| `isClosedOnOutsideClick`  | When `true`, mousedown outside the dialog calls `onClose` (subject to the opt-out below).         |
| `autoFocus`               | Forwarded to `ModalFocus`. When `true`, the first focusable child is focused on mount.            |
| `initialFocusRef`         | Forwarded to `ModalFocus`. When set, this element receives focus on mount.                        |
| `finalFocusRef`           | Forwarded to `ModalFocus`. When set, this element receives focus on close.                        |
| `restoreFocus`            | Forwarded to `ModalFocus`. When `true`, focus returns to the element that opened the modal.       |

## Outside-click opt-out for floating UI primitives

Floating UI primitives (Popover, Tooltip, Select dropdown, etc.) often render via portal as **siblings** to the modal â€” so a click inside the popover lands outside the modal's dialog DOM tree, which would otherwise trigger the close-on-outside-click handler.

To prevent this, `ModalContent` ignores any mousedown whose target's `closest('[data-mm-modal-ignore-outside-click]')` is non-null. Set the attribute on the floating element's root to opt out:

```tsx
import { MODAL_CONTENT_IGNORE_OUTSIDE_CLICK_ATTR } from '@metamask/design-system-react';

<Box {...{ [MODAL_CONTENT_IGNORE_OUTSIDE_CLICK_ATTR]: '' }}>{popoverContent}</Box>
```

The constant resolves to the literal string `data-mm-modal-ignore-outside-click`. The attribute presence is what matters â€” the value is unused.
