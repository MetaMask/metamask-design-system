# Modal

`Modal` is the root of the modal component family. When `isOpen` is `true`, it portals its `children` into `document.body` and provides a context that exposes the modal's behavior configuration (close-on-outside-click, close-on-escape, focus management) to any descendant via `useModalContext()`. When `isOpen` is `false`, the subtree is unmounted entirely so focus state and side effects are released.

`Modal` is intentionally a thin shell. Compose it with `ModalOverlay`, `ModalContent`, `ModalHeader`, `ModalBody`, and `ModalFooter` to assemble a full modal UI.

```tsx
import { Modal } from '@metamask/design-system-react';

<Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
  {/* ModalOverlay + ModalContent + … */}
</Modal>;
```

## `useModalContext`

`Modal` exposes its behavior configuration via context. Sibling/descendant components in the modal family can read it to decide how to react to clicks, keypresses, and lifecycle events:

```tsx
import { useModalContext } from '@metamask/design-system-react';

function MyCloseButton() {
  const { onClose } = useModalContext();
  return <button onClick={onClose}>Close</button>;
}
```

Calling `useModalContext()` outside of a `<Modal>` subtree throws a developer-friendly error.
