# TextFieldSearch

TextFieldSearch is a controlled search input built on top of `TextField`. It renders a leading search icon, sets the native input `type` to `search`, and shows a trailing clear button when the field has a value. Use it for filtering or search experiences; for a general-purpose text input, use `TextField`.

```tsx
import { TextFieldSearch } from '@metamask/design-system-react';

<TextFieldSearch
  placeholder="Search"
  value={value}
  onChange={(event) => setValue(event.target.value)}
  clearButtonOnClick={() => setValue('')}
/>;
```

## Props

### `showClearButton`

When `true` (default), a clear button is rendered after the input when `value` is truthy. Set to `false` to hide the clear button entirely.

<Canvas of={TextFieldSearchStories.ShowClearButton} />

### `clearButtonOnClick`

Click handler invoked when the clear button is pressed. Use it to reset the controlled `value`.

### `clearButtonProps`

Additional props forwarded to the underlying `ButtonIcon`. Use `clearButtonProps.ariaLabel` to provide a localized label (defaults to `'Clear'`).

<Canvas of={TextFieldSearchStories.ClearButtonProps} />

### `size`

Inherited from `TextField`. Controls the height of the container.

<Canvas of={TextFieldSearchStories.Size} />

### `isDisabled`

Inherited from `TextField`. Disables the inner input and the clear button container.

<Canvas of={TextFieldSearchStories.IsDisabled} />

### `isError`

Inherited from `TextField`. Renders the error border treatment.

<Canvas of={TextFieldSearchStories.IsError} />

### Other props

All other `TextField` props (`isReadOnly`, `placeholder`, `inputProps`, `inputRef`, `startAccessory`, `endAccessory`, etc.) are supported. The `type` prop is fixed to `'search'` and is not configurable.
