# ModalHeader

`ModalHeader` is the top section of a modal. It renders a semantic `<header>` with a centered title and optional back/close buttons. The component owns the iconography and size of the built-in buttons (`ArrowLeft` / `Close` at `ButtonIconSize.Md`); consumers configure the click handler and the (required) accessible label via `backButtonProps` / `closeButtonProps`.

```tsx
import { ModalHeader } from '@metamask/design-system-react';

<ModalHeader
  onClose={handleClose}
  closeButtonProps={{ ariaLabel: t('close') }}
>
  {t('removeAccount')}
</ModalHeader>;
```

## Props

### `children`

Header title content. When a string, it is auto-wrapped in a `Text` component with `TextVariant.HeadingSm` and `TextAlign.Center`. When a `ReactNode`, it is rendered as-is so consumers can supply custom title layouts.

### `onBack` + `backButtonProps`

When `onBack` is set, the component renders a back `ButtonIcon` in the start slot. `backButtonProps` is **co-required** (the type system enforces this): consumers must supply at least `ariaLabel`. The component owns `iconName` (`IconName.ArrowLeft`) and `size` (`ButtonIconSize.Md`); they cannot be overridden via the prop bag.

`ModalHeader` is i18n-agnostic â€” pass localized labels from your application's i18n layer (e.g. `t('back')`).

### `onClose` + `closeButtonProps`

Same shape as `onBack` / `backButtonProps`. When `onClose` is set, a close `ButtonIcon` (`IconName.Close`, `ButtonIconSize.Md`) renders in the end slot. `closeButtonProps.ariaLabel` is required.
