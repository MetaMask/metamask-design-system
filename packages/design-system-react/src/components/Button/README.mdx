# Button

`Button` is a labeled element that a user can click or tap to initiate an action.

```tsx
import { Button } from '@metamask/design-system-react';

<Button onClick={() => {}}>Button</Button>;
```

<Canvas of={ButtonStories.Default} />

## Props

### Variant

The Button component supports three variants:

- `ButtonVariant.Primary` (default) - High emphasis, used for primary actions
- `ButtonVariant.Secondary` - Medium emphasis, used for secondary actions
- `ButtonVariant.Tertiary` - Low emphasis, used for tertiary actions

<Canvas of={ButtonStories.Variant} />

### Size

Buttons support three sizes:

- `Sm` (32px)
- `Md` (40px)
- `Lg` (48px) - default

<Canvas of={ButtonStories.Size} />

### IsDanger

Use `isDanger` for destructive actions.

<Canvas of={ButtonStories.IsDanger} />

### IsInverse

Use `isInverse` when the button needs to be displayed on a dark background.

> NOTE: When both `isInverse` and `isDanger` are set to `true`, all button variants (Primary, Secondary, and Tertiary) will have the same inverse danger styling.

### Class Name

Use the `className` prop to add custom CSS classes to the component. These classes will be merged with the component's default classes using `twMerge`, allowing you to:

- Add new styles that don't exist in the default component
- Override the component's default styles when needed

Example:

```tsx
// Adding new styles
<Button className="mx-2 my-4">Button content</Button>
```

Note: When using `className` to override default styles, the custom classes will take precedence over the component's default classes.

### Style

The `style` prop should primarily be used for dynamic inline styles that cannot be achieved with className alone. For static styles, prefer using className with Tailwind classes.
